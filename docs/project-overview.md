# 专利纠纷中立评估平台

## 产品说明书

**版本**: 2.0 | **日期**: 2026年6月 | **状态**: MVP 迭代中

---

## 一、平台简介

专利纠纷中立评估平台是一个面向中国市场的在线争议解决系统，整合 **AI 智能分析** 与 **专家中立评估** 双轨路径，参照 WIPO 专家裁决制度及 PMAC-UPC（统一专利法院专利调解与仲裁中心）服务体系架构设计。

平台致力于为专利纠纷当事人提供 **高效、专业、保密** 的中立评估服务，将传统争议解决周期从 12-24 个月缩短至约 30 天，成本降低 30-50%。

---

## 二、核心价值

### 解决什么问题

| 现状痛点 | 平台方案 |
|----------|----------|
| 专利技术分析高度专业化，普通法官难以独立完成技术判断 | AI 预分析 + 专业领域专家终裁，双重保障 |
| 案件审理周期长（12-24 个月），时间成本高 | 全流程在线化，目标 30 天内完成评估 |
| 专利估值和许可费率确定缺乏标准化方法 | 多方法估值引擎（收益法/市场法/成本法/许可费率法） |
| 专家资源分散，匹配效率低 | 结构化专家库 + 技术领域面板分类 + 智能推荐 |
| 当事人缺乏纠纷解决路径选择指导 | ADR 评估引导系统，智能推荐最优解决路径 |

### 核心差异化优势

**1. AI + 专家双轨模式**

平台独创 AI 预分析与专家终裁相结合的双轨评估体系。AI 系统自动完成技术特征分解、侵权比对和专利估值等基础分析工作，专家在此基础上进行深度判断和最终裁决，兼顾效率与专业性。

**2. 参照国际标准的流程设计**

平台流程参照 WIPO 专家裁决规则（27条规则）及 PMAC-UPC 服务体系设计，包括独立性声明签署（第10条）、禁止单方沟通原则（第14条）、裁决30日内更正机制（第17条）等，确保程序的规范性和公信力。

**3. 多元争议解决路径**

除传统的专家裁决外，平台提供早期中立评估（ENE）、ADR 评估引导等多元路径，帮助当事人根据纠纷特点选择最适合的解决方式，后续还将接入调解和仲裁通道。

**4. 全流程数字化**

从案件创建、材料上传、AI 分析、专家评估到裁决出具，全流程在线完成，配套完整的审计日志和消息通知体系。

## 三、用户角色

平台服务三类用户群体：

### 当事人（申请方/被申请方）

专利权人、被控侵权方、企业法务及专利代理人。可在线提交评估申请、上传案件材料、查看 AI 分析报告、接收评估意见，全程掌握案件进度。

### 评估专家

专利律师、技术专家、行业权威。接受平台指派，利用 AI 分析报告辅助，对专利纠纷进行专业评估并出具正式裁决意见。平台提供独立的工作台和完整的案件管理工具。

### 平台管理员

负责案件流程管理、专家资源调配、系统运营监控。通过管理后台实现全量案件的状态流转控制和专家匹配。

## 四、核心功能

### 4.1 纠纷评估引导

不确定选择哪种纠纷解决方式？平台提供免费的在线评估引导服务。

当事人只需回答关于纠纷类型、专利信息、争议金额、技术复杂度等几个简单问题，系统即通过决策树算法智能推荐最适合的纠纷解决路径：

- **专家评估** — 适合高价值、高技术复杂度的侵权纠纷
- **早期中立评估（ENE）** — 适合双方愿意配合的纠纷，促进快速和解
- **调解** — 适合小额纠纷或曾有协商基础的案件
- **仲裁** — 适合需要约束力裁决的复杂案件

评估结果可直接跳转案件创建，相关字段自动预填，减少重复录入。

### 4.2 案件管理

完整的案件生命周期管理：

```
创建案件 → 上传材料 → 提交 → 被申请人回复 → 受理 → 专家指派 → 评估 → 裁决 → 完成
```

**案件创建**：3 步向导式操作 — 填写基本信息、上传材料、确认提交。支持从评估引导结果自动导入。

**被申请人参与**：案件提交后系统自动通知被申请人，设有 15 天回复期限。被申请人可选择接受评估、拒绝参与或提出反请求，充分保障双方程序权利。

**文档管理**：支持专利证书、权利要求书、证据材料、技术说明等分类上传，单文件最大 50MB，支持 PDF、Word、图片格式。

**状态追踪**：当事人可通过控制台实时查看案件状态、回复截止日期、专家指派情况，系统消息自动推送关键节点通知。

### 4.3 AI 智能分析

平台整合 AI 评估引擎，为专家提供结构化的技术分析参考：

**权利要求解释**
- 自动提取独立权利要求和从属权利要求
- 技术特征逐项分解
- 生成结构化解释报告

**侵权比对分析**
- 被控产品/方法与权利要求的逐特征比对
- 字面侵权分析
- 等同原则分析
- 置信度评分与推理过程可追溯

**专利估值分析**
- 收益法：基于未来收益现值（DCF 模型）
- 市场法：参考可比交易案例
- 成本法：基于研发投入与重置成本
- 许可费率法：参考行业标准费率
- 综合推荐估值区间及敏感性分析

AI 分析结果以结构化报告形式呈现，专家可查看并添加批注，最终由专家独立作出判断。

### 4.4 早期中立评估（ENE）

早期中立评估是参照国际最佳实践设计的独立评估模块，旨在通过中立专家的专业意见促进当事人和解。

**灵活的评估范围**
- 仅侵权分析：聚焦侵权是否成立
- 仅估值分析：聚焦专利价值和许可费率
- 全面评估：涵盖侵权和估值的综合分析

**约束性选择**
当事人可事先约定评估意见是否具有约束力。非约束性意见供双方参考协商，约束性意见具有等同裁决的效力。

**两阶段意见**
- 初步评估意见：在全面审查材料后出具初步专业判断，供当事人评估和解可能性
- 最终评估意见：综合双方材料和初步意见反馈，出具正式评估结论

**双方同意机制**
ENE 程序启动需双方当事人均确认同意参与，确保程序的自愿性和公正性。

### 4.5 专家中立人管理

**专家入驻**
具备相关资质的专业人士可在线提交入驻申请，经管理员审核通过后获得平台专家身份，接受案件指派。支持专利代理人、律师、教授、行业工程师等多元背景。

**面板分类**
专家按技术领域划分面板：电子/通信、软件/互联网、机械/制造、化学/材料、医药/生物等。系统根据案件技术领域智能推荐匹配专家。

**利益冲突管理**
系统维护专家与当事人之间的利益冲突记录（同一公司、曾代理关系、亲属关系、经济利益关系等），在专家指派前自动检查并预警，保障评估的独立性和公正性。

**评价体系**
案件完成后，当事人可从专业性、公正性、时效性、技术水平等维度对专家进行评分。评分结果纳入专家档案，作为后续案件指派的参考依据。

### 4.6 裁决管理

**裁决类型**
- 临时裁决：对部分争议焦点出具的阶段性意见
- 最终裁决：对全案的正式评估结论

**裁决更正**
裁决出具后 30 日内，专家可对笔误或计算错误进行更正（参照 WIPO 第17条规则），更正记录保留可追溯。

**通知机制**
裁决出具后系统自动通知双方当事人，确保信息及时送达。

### 4.7 沟通与审计

**站内消息系统**：当事人与专家之间可通过平台进行案件相关沟通，所有消息记录关联案件，可追溯。

**完整审计日志**：系统记录所有关键操作（案件创建、状态变更、专家指派、裁决出具等），确保全流程可审计。

**保密管理**：案件信息仅对相关当事人、指派专家和管理员可见，AI 分析报告标记为保密信息，裁决结果默认保密。

## 五、技术架构

### 技术选型

| 层级 | 技术方案 |
|------|----------|
| 前端框架 | Next.js 16 (App Router) + React 19 |
| UI 组件 | Ant Design 6 + Tailwind CSS 4 |
| 后端逻辑 | Next.js Server Actions |
| 数据库 | SQLite（开发）/ PostgreSQL（生产） |
| ORM | Prisma 7 |
| 认证 | JWT (jose) + httpOnly Cookie |
| AI 服务 | Python FastAPI 微服务（ArbitralAgent 框架） |

### 系统特点

- **全栈 TypeScript**：前后端统一语言，类型安全保障
- **Server Actions**：无需单独 API 层，服务端直接执行业务逻辑
- **Server/Client 组件分离**：服务端负责认证和数据获取，客户端处理交互
- **角色隔离**：三套独立门户（当事人/专家/管理员），独立布局和导航
- **AI 服务解耦**：AI 分析作为独立微服务部署，支持渐进式集成和 mock 降级

### 数据规模

- 16 个数据模型，覆盖案件、用户、专家、评估、裁决、消息、审计全链路
- 12 个 Server Action 文件，37 个导出函数
- 79 个源文件，覆盖 3 个角色门户 + 2 个认证页面 + 2 个 API 路由

## 六、典型使用场景

### 场景一：5G 专利侵权评估

深圳华芯科技发现北京创新电子的 5G 基站产品使用了其专利技术。通过平台评估引导确认适合专家评估路径，在线提交案件和侵权证据。平台 AI 系统自动完成权利要求技术特征分解和侵权比对分析，清华大学通信领域专家在此基础上出具评估意见，认定侵权可能性较大，建议双方协商许可费率。全程耗时约 25 天。

### 场景二：AI 图像识别专利 ENE

两家企业在 AI 图像识别专利方面存在争议，双方均愿意配合评估。通过平台发起早期中立评估程序，选择全面评估范围和非约束性意见模式。双方确认同意后，指派的专家先出具初步评估意见，双方据此进行和解协商，最终达成许可协议。

### 场景三：半导体专利估值

某企业需要确定其半导体封装专利的合理许可费率。通过平台提交估值评估申请，AI 系统采用收益法、市场法、成本法和许可费率法四种方法进行综合估值，专家审核后出具最终估值意见，为许可谈判提供量化依据。
