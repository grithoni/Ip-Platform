## 执行摘要

专利纠纷中立评估平台是一个面向中国市场的在线争议解决系统，整合 AI 智能评估与专家裁决两条路径，参照 WIPO 专家裁决制度架构设计。平台旨在为专利纠纷当事人提供高效、专业、保密的中立评估服务，降低传统争议解决的时间和成本。

核心价值主张：
- **AI 智能评估**：利用 ArbitralAgent 框架，自动进行权利要求解释、侵权比对和专利估值分析
- **专家裁决流程**：参照 WIPO 27 条规则，实现从案件提交到裁决出具的全流程数字化管理
- **多角色协作**：支持当事人、专家、管理员三类用户角色协同工作

---

## 问题陈述

**现状**：中国专利纠纷解决主要依赖传统诉讼和仲裁，存在以下痛点：
- 专利技术分析高度专业化，普通法官/仲裁员难以独立完成技术判断
- 案件审理周期长（通常 12-24 个月），当事人时间成本高
- 专利估值和许可费率确定缺乏标准化方法
- 专家资源分散，匹配效率低

**解决方案**：构建在线中立评估平台，通过 AI 预分析 + 专家终裁的模式，提供快速、专业的专利纠纷评估服务。

**业务影响**：
- 将初步评估时间从数周缩短至数天
- 降低当事人争议解决成本 30-50%
- 提高专家资源利用效率
- 建立标准化的专利评估方法论

## 成功指标

**主要 KPI：**
- **案件处理效率**：平均案件周期 ≤ 30 天（从提交到裁决），目标 6 个月内达成
- **AI 评估准确率**：权利要求分析准确率 ≥ 85%（与专家判断对比），目标 9 个月内达成
- **用户满意度**：当事人满意度评分 ≥ 4.2/5.0，目标 12 个月内达成
- **专家利用率**：专家平均每月处理案件数 ≥ 3 件，目标 6 个月内达成
- **系统可用性**：平台可用率 ≥ 99.5%，持续监控

**验证方法**：
- 案件周期：通过案件管理系统自动统计
- AI 准确率：随机抽样 100 件案件，对比 AI 评估与专家终裁结果
- 用户满意度：案件结束后发送满意度调查问卷
- 专家利用率：每月统计专家工作量
- 系统可用性：使用 uptime 监控工具

## 用户画像

### 主要用户：当事人（申请方/被申请方）
- **角色**：专利权人、被控侵权方、企业法务、专利代理人
- **目标**：快速获得专业的专利纠纷评估意见，降低争议解决成本
- **痛点**：传统诉讼周期长、成本高；缺乏技术背景难以理解专利分析
- **技术水平**：中等，熟悉基本在线操作，但不一定了解专利法律细节
- **典型场景**：企业 A 认为企业 B 的产品侵犯其专利权，希望通过中立评估快速确定是否存在侵权及合理许可费率

### 次要用户：专家/裁决人
- **角色**：专利律师、技术专家、专利代理人、行业专家
- **目标**：高效处理评估案件，获得合理报酬，建立专业声誉
- **痛点**：案件材料整理耗时；缺乏标准化分析工具；与当事人沟通效率低
- **技术水平**：高，熟悉专利分析和法律程序
- **典型场景**：专利律师接受平台指派，对一起专利侵权纠纷进行技术分析和估值评估

### 管理用户：平台管理员
- **角色**：平台运营人员、案件管理员
- **目标**：确保案件流程顺畅，合理分配专家资源，维护平台秩序
- **痛点**：案件分配需要考虑专家资质匹配；费用结算复杂；需要监控案件进度
- **技术水平**：高，熟悉后台管理系统操作
- **典型场景**：管理员收到新案件申请，根据专利技术领域和案件复杂程度匹配合适的专家

## 用户故事与验收标准

### 故事 1：当事人提交专利纠纷评估申请

**作为** 当事人
**我想要** 在线提交专利纠纷评估申请
**以便** 启动中立评估程序，获得专业评估意见

**验收标准：**
- [ ] 用户可以注册/登录平台（支持手机号、邮箱）
- [ ] 用户可以创建新案件，填写基本信息（专利号、纠纷类型、争议金额等）
- [ ] 用户可以上传案件材料（专利证书、权利要求书、被控侵权产品说明等，支持 PDF/Word/图片）
- [ ] 系统自动生成案件编号，并显示在案件详情页
- [ ] 用户可以查看案件状态（待审核/已分配专家/评估中/已完成）
- [ ] 材料上传支持批量操作，单个文件不超过 50MB
- [ ] 必填字段校验失败时，显示明确的错误提示

### 故事 2：AI 进行权利要求分析

**作为** AI 评估系统
**我想要** 自动分析专利权利要求
**以便** 为专家提供初步的技术分析参考

**验收标准：**
- [ ] 系统自动提取专利权利要求书中的独立权利要求和从属权利要求
- [ ] AI 生成权利要求解释报告，包括技术特征分解
- [ ] AI 进行侵权比对分析，识别被控侵权产品/方法与权利要求的对应关系
- [ ] 分析报告以结构化格式展示（技术特征、比对结果、初步结论）
- [ ] 专家可以查看 AI 分析报告并添加批注
- [ ] AI 分析过程可追溯，显示分析依据和推理过程
- [ ] 分析结果支持导出为 PDF

### 故事 3：AI 进行专利估值分析

**作为** AI 评估系统
**我想要** 评估专利价值和合理许可费率
**以便** 为裁决提供量化参考依据

**验收标准：**
- [ ] 系统根据专利类型、技术领域、剩余有效期等因素计算基础估值
- [ ] AI 采用收益法、市场法、成本法等多种方法进行估值
- [ ] 系统提供可比交易案例检索（基于专利分类号、技术领域等）
- [ ] AI 生成许可费率建议，参考行业标准和可比案例
- [ ] 估值报告包含假设条件、计算过程和敏感性分析
- [ ] 专家可以调整估值参数并重新计算
- [ ] 估值结果以图表形式可视化展示

### 故事 4：专家进行裁决

**作为** 专家/裁决人
**我想要** 查看案件材料和 AI 分析报告，出具裁决意见
**以便** 完成专家裁决程序，为当事人提供具有约束力的裁决

**验收标准：**
- [ ] 专家可以查看案件全部材料（当事人提交的文件、AI 分析报告）
- [ ] 专家可以添加批注、标记关键信息
- [ ] 专家可以要求当事人补充材料（通过平台消息系统）
- [ ] 专家可以出具临时/部分裁决或最终裁决
- [ ] 裁决书模板包含：事项描述、分析过程、裁决理由、裁决结果
- [ ] 裁决书支持在线编辑和预览
- [ ] 裁决提交后，系统自动通知双方当事人
- [ ] 裁决生效后 30 天内，专家可以更正笔误

### 故事 5：管理员管理案件和专家

**作为** 平台管理员
**我想要** 管理案件流程和专家资源
**以便** 确保评估程序高效运行

**验收标准：**
- [ ] 管理员可以查看所有案件列表，支持按状态、时间、类型筛选
- [ ] 管理员可以为案件分配专家（手动或系统推荐）
- [ ] 系统根据专利技术领域、案件复杂程度、专家可用性推荐匹配专家
- [ ] 管理员可以查看专家工作量和可用状态
- [ ] 管理员可以监控案件进度，对超时案件进行提醒
- [ ] 管理员可以管理费用（管理费收取、专家费用结算、预付金管理）
- [ ] 系统生成案件统计报表（案件数量、处理时长、专家工作量等）

## 功能需求

### 核心功能模块

#### 模块 1：案件提交与管理

**功能描述**：支持当事人在线提交专利纠纷评估申请，管理案件全生命周期。

**用户流程**：
1. 当事人注册/登录平台
2. 创建新案件，填写基本信息（专利信息、纠纷类型、争议金额、当事人信息）
3. 上传案件材料（专利证书、权利要求书、技术说明、被控侵权证据等）
4. 系统生成案件编号，案件状态变为"待审核"
5. 管理员审核案件材料完整性
6. 案件进入专家分配环节
7. 当事人可以随时查看案件进度

**案件状态流转**：
```
草稿 → 待审核 → 已受理 → 专家分配中 → 评估中 → 裁决中 → 已完成
                  ↓
              材料不全（退回补充）
```

**数据模型**：
- Case（案件）：id, case_number, patent_number, patent_title, dispute_type, amount_in_dispute, status, created_at, updated_at
- Party（当事人）：id, case_id, user_id, role (applicant/respondent), company_name, contact_info
- Document（材料）：id, case_id, uploader_id, file_name, file_path, file_type, upload_time, category

**边界情况**：
- 同一专利号重复提交：提示用户已有相关案件，询问是否继续
- 文件格式不支持：显示支持的格式列表（PDF, DOC, DOCX, JPG, PNG）
- 文件大小超限：提示单个文件不超过 50MB，建议压缩或分割

#### 模块 2：AI 权利要求分析

**功能描述**：利用 ArbitralAgent 自动分析专利权利要求，生成侵权比对报告。

**用户流程**：
1. 案件受理后，系统自动触发 AI 分析
2. AI 提取专利权利要求书内容
3. AI 分析被控侵权产品/方法描述
4. 生成技术特征分解表
5. 进行侵权比对分析
6. 生成结构化分析报告
7. 专家查看报告并可添加批注

**AI 分析输出结构**：
```json
{
  "claim_interpretation": {
    "independent_claims": [...],
    "dependent_claims": [...],
    "technical_features": [...]
  },
  "infringement_analysis": {
    "feature_comparison": [...],
    "literal_infringement": {...},
  }
}
```

**与 ArbitralAgent 集成**：
- Next.js 后端通过 HTTP API 调用 ArbitralAgent Python 服务
- ArbitralAgent 运行在独立端口（默认 8000）
- API 端点：`POST /api/agent/analyze-claims`
- 请求体：专利文本、被控侵权产品描述
- 响应体：结构化分析报告（JSON）

#### 模块 3：AI 估值分析

**功能描述**：基于多种估值方法评估专利价值和合理许可费率。

**估值方法**：
1. **收益法**：基于专利未来收益现值计算
2. **市场法**：参考可比交易案例
3. **成本法**：基于研发投入和重置成本
4. **许可费率法**：基于行业标准许可费率

**AI 估值输出结构**：
```json
{
  "valuation_methods": {
    "income_approach": {...},
    "market_approach": {...},
    "cost_approach": {...},
    "royalty_rate": {...}
  },
  "comparable_cases": [...],
  "recommended_range": {
    "low": 0,
    "high": 0,
    "recommended": 0
  },
  "assumptions": [...],
  "sensitivity_analysis": {...}
}
```

**与 ArbitralAgent 集成**：
- API 端点：`POST /api/agent/valuate-patent`
- 请求体：专利信息、技术领域、市场数据
- 响应体：估值报告（JSON）

#### 模块 4：专家裁决流程

**功能描述**：参照 WIPO 规则实现专家裁决全流程管理。

**流程步骤**（参照 WIPO 规则）：

**第 1 步：专家指定（第 9 条）**
- 系统根据专利技术领域、案件复杂程度推荐专家候选人
- 当事人可在 7 天内对专家候选人提出异议
- 专家接受指定前需签署公正性和独立性声明（第 10 条）

**第 2 步：程序进行（第 14 条）**
- 专家查看案件材料和 AI 分析报告
- 专家可要求当事人补充材料
- 专家可通过平台与当事人沟通（禁止单方沟通，第 14 条 c 款）
- 专家可召开在线会议（视频/音频）

**第 3 步：裁决作出（第 17 条）**
- 专家出具书面裁决
- 裁决包含：事项描述、分析过程、裁决理由、裁决结果
- 裁决提交给平台，平台正式通知当事人

**第 4 步：裁决更正**
- 裁决生效后 30 天内，可请求更正笔误（第 17 条）

**专家回避机制（第 11 条）**：
- 当事人可在得知指定后 7 天内申请专家回避
- 回避理由：存在对专家公正性或独立性合理怀疑的情况
- 如对方不同意，由平台管理员作出最终决定

**数据模型**：
- Expert（专家）：id, user_id, name, technical_fields, qualifications, availability, hourly_rate
- ExpertAssignment（专家分配）：id, case_id, expert_id, assigned_at, status, declaration_signed
- Determination（裁决）：id, case_id, expert_id, type (interim/final), content, issued_at, corrected_at

## 技术约束

### 性能要求
- **API 响应时间**：普通 API 调用 < 200ms，AI 分析接口 < 30s（可异步处理）
- **并发支持**：MVP 阶段支持 50 并发用户
- **文件上传**：支持最大 50MB 单文件，最大 500MB 总文件
- **AI 分析**：单个案件分析时间 < 10 分钟

### 安全要求
- **认证**：NextAuth.js 支持手机号、邮箱登录
- **授权**：基于角色的访问控制（RBAC），确保当事人只能查看自己的案件
- **数据加密**：敏感数据（案件材料、裁决书）加密存储
- **保密性**（参照 WIPO 第 16 条）：
  - 案件信息仅对当事人、专家、管理员可见
  - AI 分析报告标记为保密信息
  - 裁决结果默认保密，除非当事人另有约定
- **审计日志**：记录所有关键操作（案件创建、材料上传、裁决提交等）

### 集成要求
- **ArbitralAgent**：独立 Python 服务，通过 HTTP API 集成
  - 服务地址：`http://localhost:8000`（开发环境）
  - API 文档：Swagger/OpenAPI
  - 超时设置：30 秒
  - 重试机制：失败后重试 3 次
- **文件存储**：本地文件系统（MVP），后续可迁移至对象存储（如阿里云 OSS）
- **邮件通知**：集成邮件服务（如 SendGrid 或阿里云邮件推送）

### 技术栈
- **前端**：Next.js 16 + TypeScript + Ant Design 5 + Tailwind CSS
- **后端**：Next.js API Routes（App Router）
- **数据库**：SQLite + Prisma ORM（开发），后续可迁移至 PostgreSQL
- **AI 服务**：ArbitralAgent（Python，独立服务）
- **认证**：NextAuth.js
- **状态管理**：Zustand
- **图表**：Ant Design Charts 或 Recharts

## MVP 范围与分阶段交付

### 第一阶段：MVP（初始上线）

**MVP 定义**：实现从案件提交到裁决出具的核心流程，验证 AI + 专家协作模式的可行性。

**核心功能**：
1. **用户系统**：注册/登录（手机号+验证码）、个人中心
2. **案件管理**：创建案件、上传材料、查看案件列表和详情、案件状态跟踪
3. **AI 权利要求分析**：集成 ArbitralAgent，自动生成权利要求解释和侵权比对报告
4. **AI 估值分析**：集成 ArbitralAgent，生成专利估值和许可费率建议
5. **专家裁决流程**：专家分配、查看材料、出具裁决、裁决通知
6. **管理员后台**：案件管理、专家管理、基础统计

**MVP 不包含**：
- 在线支付和费用管理
- 在线视频会议
- 多语言支持
- 移动端适配
- 复杂的专家匹配算法（先手动分配）

**预计工期**：8-10 周

### 第二阶段：增强功能（上线后迭代）

**增强内容**：
1. **费用管理**：管理费收取、专家费用结算、预付金管理
2. **在线沟通**：集成视频会议（如腾讯会议 API）、即时消息
3. **专家智能匹配**：基于技术领域、案件复杂度、专家可用性的智能推荐算法
4. **裁决更正**：30 天内裁决更正流程
5. **保密信息管理**：保密信息标记、分级披露、保密承诺书签署

**预计工期**：6-8 周

### 第三阶段：生产就绪（长期规划）

**生产化内容**：
1. **数据库迁移**：SQLite → PostgreSQL
2. **文件存储迁移**：本地 → 阿里云 OSS
3. **部署优化**：Docker 容器化、CI/CD 流水线
4. **安全加固**：数据加密、渗透测试、安全审计
5. **性能优化**：缓存策略、CDN、负载均衡
6. **移动端**：响应式设计或原生 App

## 依赖与阻塞项
